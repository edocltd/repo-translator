# Аналіз помилок: 48 реальних проблем перекладу репозиторіїв

Кожна проблема задокументована на основі реального досвіду перекладу [luongnv89/claude-howto](https://github.com/luongnv89/claude-howto) (22K+ ⭐). Кожен запис містить: сценарій, рівень ризику та реалізований захист.

---

## Категорія 1: Сканування та аналіз репо

### P1.1: Великий репозиторій
**Сценарій**: Користувач запускає інструмент на монорепо з 50 000+ файлів.
**Захист**: Ліміт файлів (за замовчуванням: 500), ліміт глибини (10 рівнів), попередження з порадою звузити область.

### P1.2: Репозиторій без git
**Сценарій**: Каталог без `.git/`.
**Захист**: Інструмент працює без git. Sync-check вимкнено з попередженням. Використовує хеш файлу замість SHA коміту.

### P1.3: Наявна структура перекладів
**Сценарій**: Репо вже має каталоги `uk/`, `translations/`, `i18n/`.
**Захист**: Автовиявлення наявних перекладів та виключення зі сканування. Звіт про знайдене.

### P1.4: Символічні посилання та циклічні посилання
**Захист**: Не переходити за симлінками. Відстеження відвіданих шляхів через `set(realpath)`.

### P1.5: Бінарні файли з розширенням .md
**Захист**: Перевірка перших 8 КБ на null-байти. Якщо >1% нулів → класифікувати як бінарний, пропустити.

### P1.6: Порожні .md файли
**Захист**: Пропустити файли <10 байт. Пропустити файли з лише фронтматером (без прози).

### P1.7: Однакові назви файлів у різних каталогах
**Захист**: Завжди зберігати повну структуру каталогів. Ніколи не зводити до плоскої структури.

## Категорія 2: Якість перекладу (вивід AI)

### P2.1: AI скорочує файл ⚠️ КРИТИЧНО
**Реальний випадок**: Оригінал 1945 рядків, AI повернув 540 (58% втрачено).
**Захист**: Перевірка кількості рядків після перекладу. Відхилити якщо <85% оригіналу.

### P2.2: AI перекладає код-блоки ⚠️ КРИТИЧНО
**Реальний випадок**: `mkdir -p .claude/commands` стало `створити_каталог .claude/commands`.
**Захист**: Витягти код-блоки як плейсхолдери ПЕРЕД відправкою AI. Відновити ПІСЛЯ. Код ніколи не проходить через AI.

### P2.3: AI перекладає мітки Mermaid
**Захист**: Обробляти Mermaid-блоки як код-блоки (витягнення плейсхолдерів).

### P2.4: AI змінює Markdown-форматування
**Захист**: Валідація кількості заголовків, рядків таблиць, маркерів bold/italic відповідно до оригіналу.

### P2.5: AI вигадує URL
**Захист**: Витягти URL з оригіналу та перекладу. Звітувати про додані/видалені URL.

### P2.6: AI видаляє HTML-теги
**Захист**: Витягти HTML як плейсхолдери (як код-блоки).

### P2.7: AI додає примітки перекладача
**Захист**: Правило промпту: "НЕ додавайте примітки." Валідація: пошук патернів "note:", "translator".

### P2.8: Великий файл перевищує контекст ⚠️ КРИТИЧНО
**Реальний випадок**: 87 КБ / 3136 рядків файл.
**Захист**: Розбивка за заголовками `##`. Якщо секція все ще занадто велика → розбивка за `###`. Кожен фрагмент отримує той самий глосарій та правила.

### P2.9: AI повертає обрізану відповідь
**Захист**: Перевірка чи останній рядок — повне речення. Перевірка чи кількість код-блоків парна (усі блоки закриті).

## Категорія 3: Посилання та якорі

### P3.1: Зламані якорі після перекладу ⚠️ КРИТИЧНО
**Реальний випадок**: `[See below](#slash-commands)` але заголовок тепер `## Слеш-команди` → якір `#слеш-команди`.
**Захист**: Після перекладу зібрати всі заголовки → згенерувати якорі → порівняти з посиланнями → автовиправити невідповідності.

### P3.2: Варіанти апострофів у якорях ⚠️ КРИТИЧНО
**Реальний випадок**: Заголовок використовує `'` (U+0027), який видаляється з якоря, але посилання використовує `ʼ` (U+02BC), який залишається → невідповідність.
**Захист**: Нормалізація всіх варіантів апострофів при порівнянні якорів.

### P3.3: Дублікати заголовків
**Реальний випадок**: Два заголовки `## Prompt-хуки`. GitHub додає суфікс `-1` до другого. Валідатор не обробляв це.
**Захист**: Відстеження лічильника заголовків. Генерація суфіксів `-1`, `-2` для дублікатів.

### P3.4: Відносні посилання на неперекладені файли
**Сценарій**: `translations/uk/docs/guide.md` посилається на `../src/deploy.sh`, якого не існує за цим шляхом.
**Захист**: Переписування відносних посилань: якщо ціль перекладена → залишити локальне посилання. Якщо ні → переписати на оригінал.

### P3.5: Абсолютні шляхи
**Захист**: Виявити та попередити. Не переписувати автоматично (специфічна поведінка фреймворку).

### P3.6: Міжфайлові якорні посилання
**Сценарій**: `[See](other.md#section)` де `other.md` перекладено → `#section` може стати `#секція`.
**Захист**: Спочатку визначити цільовий файл, потім перевірити якорі у правильній версії.

## Категорія 4: Кодування та формат файлів

### P4.1: Файл не UTF-8 ⚠️ КРИТИЧНО
**Реальний випадок**: AI повернув частково CP1251 у UTF-8 файлі. 240 некоректних байт.
**Захист**: Перевірка UTF-8 після кожного запису. Авторемонт: виявлення поганих діапазонів байт, спроба перекодування CP1251 → UTF-8.

### P4.2: BOM (Byte Order Mark)
**Захист**: Ігнорувати BOM при читанні. Ніколи не записувати BOM.

### P4.3: CRLF vs LF
**Захист**: Відповідати закінченням рядків оригіналу. За замовчуванням LF.

### P4.4: Відсутній завершальний перенос рядка
**Захист**: Завжди додавати `\n` в кінці файлу.

### P4.5: Невидимі Unicode-символи
**Захист**: Видалення U+200B (пробіл нульової ширини), U+200C, U+200D, U+200E, U+200F після перекладу.

## Категорія 5: Мовно-специфічні проблеми

### P5.1: RTL-мови (арабська, іврит)
**Захист**: Попередження при ініціалізації. Обмежена підтримка Markdown для RTL.

### P5.2: CJK-мови (китайська, японська, корейська)
**Захист**: Коригування толерантності кількості рядків (CJK-текст коротший). Використання кількості код-блоків як основної структурної перевірки.

### P5.3: Спеціальний регістр символів (турецькі İ/ı)
**Захист**: Використання Unicode-коректного порівняння (`str.casefold()`).

### P5.4: Глосарій для кожної мови
**Захист**: Глосарій генерується для кожної мови. Вбудована база термінів для 10-15 популярних мов.

---

## Категорія 6: Фронтматер та метадані

### P6.1: YAML-фронтматер ламається при перекладі
**Захист**: Парсинг YAML окремо. Переклад лише поля `description`. Завжди обгортати значення лапками після перекладу.

### P6.2: Подвійний фронтматер
**Захист**: Тільки перший блок `---...---` на початку файлу є фронтматером.

### P6.3: Блоки з тильдою (`~~~~`)
**Захист**: Обробка як ` ``` `, так і `~~~~` як маркерів код-блоків.

---

## Категорія 7: Файлова система

### P7.1: Немає прав на запис
**Захист**: Перевірка доступу на запис перед початком.

### P7.2: Диск заповнений
**Захист**: Оцінка необхідного простору (2× розмір вихідних .md). Перевірка вільного місця.

### P7.3: Довгі шляхи (ліміт Windows 260 символів)
**Захист**: Попередження якщо результуючий шлях >250 символів.

### P7.4: Спеціальні символи в назвах файлів
**Захист**: Зберігати оригінальні назви. Використовувати об'єкти `Path`, а не конкатенацію рядків.

### P7.5: Чутливість до регістру
**Захист**: Використовувати точні оригінальні назви файлів. Попереджувати про дублікати лише за регістром.

---

## Категорія 8: Git та контроль версій

### P8.1: Величезний diff
**Захист**: Рекомендація розбити на пакетні PR за пріоритетом.

### P8.2: Конфлікти злиття
**Захист**: Кожна мова — окремий підкаталог → немає конфліктів між мовами.

### P8.3: Squash merge втрачає трекінг SHA
**Захист**: Резервний варіант — пошук за датою коли SHA не знайдено в історії.

---

## Категорія 9: Конфігурація

### P9.1: Некоректний YAML у конфізі
**Захист**: Валідація при завантаженні. Повернення до значень за замовчуванням з попередженням.

### P9.2: Невідомий код мови
**Захист**: Валідація за ISO 639-1. Нечітке співставлення: "ukr" → "Можливо, ви мали на увазі uk?"

### P9.3: Конфлікти глосарію
**Захист**: Перевірка на конфліктні переклади кореневих слів при завантаженні. Попередження.

---

## Категорія 10: Граничні випадки

### P10.1: Файл що складається лише з таблиць
**Захист**: Обробка таблиць рядок за рядком. Перевірка кількості `|` на рядок відповідно до оригіналу.

### P10.2: Markdown з вбудованим HTML
**Захист**: HTML-теги → плейсхолдери. Переклад тексту між тегами, не атрибутів.

### P10.3: Формули LaTeX
**Захист**: Виявлення `$...$` та `$$...$$`. Витягнення як плейсхолдерів.

### P10.4: Emoji у заголовках
**Захист**: Видалення emoji при генерації якорів (відповідає поведінці GitHub).

### P10.5: Файл без заголовків
**Захист**: Розбивка за порожніми рядками замість `##`. Якщо <4000 токенів → перекласти весь файл.

### P10.6: Дублікати метаданих i18n
**Захист**: Перевірка наявних метаданих перед додаванням. Оновити, а не дублювати.

### P10.7: Переклад перекладів ⚠️ КРИТИЧНО
**Захист**: ЗАВЖДИ виключати `translations_dir` зі сканування. Перевірка що джерело ≠ цільовий каталог.

### P10.8: Рекурсивне включення/виключення
**Захист**: `exclude: ["**/CHANGELOG.md"]` виключає в будь-якому каталозі. `exclude: ["CHANGELOG.md"]` тільки в корені.

## Підсумок захисних механізмів

### Автоматичні (без дій користувача):
| Захист | Коли | Дія |
|--------|------|-----|
| Перевірка UTF-8 | Після кожного запису | Авторемонт або відхилення |
| Плейсхолдери код-блоків | Під час перекладу | Код ніколи не проходить через AI |
| Перевірка кількості рядків | Після перекладу | Відхилити якщо <85% |
| Виправлення якорів | Після перекладу | Автозаміна зламаних якорів |
| Переписування посилань | Під час створення файлу | Автопереписування відносних шляхів |
| Завершальний перенос рядка | При записі | Автододавання |
| Видалення невидимого Unicode | Після перекладу | Автовидалення |
| Дедуплікація метаданих | При додаванні заголовка | Оновлення замість дублювання |

### Попередження (потребують уваги користувача):
| Захист | Коли | Повідомлення |
|--------|------|-------------|
| Великий репо | При скануванні | "12 847 файлів, рекомендуємо звузити область" |
| Наявні переклади | При ініціалізації | "Знайдено uk/, імпортувати?" |
| Невідповідність глосарію | При валідації | "hook перекладено як гачок (1x)" |
| RTL-мова | При ініціалізації | "Потребує додаткової уваги" |
| Абсолютні шляхи | При скануванні | "Неможливо автоматично переписати" |

### Блокуючі (зупинка процесу):
| Захист | Коли | Причина |
|--------|------|---------|
| Файл скорочено >15% | Після перекладу | AI втратив контент |
| Невідповідність кількості код-блоків | Після перекладу | Структура зламана |
| Джерело = цільовий каталог | При ініціалізації | Перекладатиме переклади |
