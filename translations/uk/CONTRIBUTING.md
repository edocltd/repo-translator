# Внесок у repo-translator

Дякуємо за інтерес до участі. Цей посібник описує процес та стандарти для внесення змін.

## Типи внесків

- **Виправлення помилок** — Виправлення проблем зі скануванням, валідацією або якорями
- **Нові перевірки валідації** — Додавання перевірок, що виявляють реальні проблеми перекладу
- **Підтримка мов** — Додавання глосаріїв, тестування на нових мовах, виправлення мовних проблем
- **Документація** — Покращення пояснень, додавання прикладів, виправлення помилок
- **Переклади** — Переклад документації проєкту за допомогою самого інструменту

## Початок роботи

```bash
git clone https://github.com/edocltd/repo-translator.git
cd repo-translator
git checkout -b your-branch-name
```

Зовнішні залежності не потрібні для основних скриптів. Опціонально: `pip install pyyaml` для підтримки глосаріїв.

## Стандарти коду

### Python

- Python 3.10+ (використовуйте `list[str]` замість `List[str]`, `str | None` замість `Optional[str]`)
- Функції мають docstrings з поясненням призначення
- Без невикористаних імпортів або мертвого коду
- Без хардкоджених шляхів — використовуйте об'єкти `Path`
- Обробляйте помилки явно (`try/except` з конкретними винятками)

### Без порожнього коду

- Без функцій-заглушок
- Без закоментованого коду
- Без `pass` окрім абстрактних методів
- Без TODO-коментарів без прив'язаного issue

## Повідомлення комітів

Дотримуйтесь conventional commits:

```
type(scope): description
```

Типи: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`

## Процес Pull Request

1. Створіть гілку з описовою назвою
2. Зробіть цілеспрямовані зміни (одна фіча або виправлення на PR)
3. Протестуйте хоча б на одному реальному репозиторії
4. Оновіть CLAUDE.md якщо поведінка змінилась
5. Подайте PR з чітким описом

## Ліцензія

Роблячи внесок, ви погоджуєтесь що ваші внески ліцензуються під MIT.
